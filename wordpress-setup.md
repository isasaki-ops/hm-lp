## 1. 年月ベースフォルダをオフにする

WordPress のメディアは通常「2025/05/」のように年月フォルダに保存されます。  
このLPでは画像パスを `/wp-content/uploads/ファイル名` で固定しているため、年月フォルダをオフにする必要があります。

1. WordPress 管理画面にログインする
2. 左メニューから **設定 → メディア** をクリックする
3. 「アップロードしたファイルを以下のフォルダに整理する」の**チェックを外す**
4. 「変更を保存」ボタンをクリックする

> **注意:** すでに年月フォルダで画像がある場合、既存の画像パスは変わりません。これからアップロードするファイルのみに適用されます。

## 3. 追加CSS を貼り付ける

LP専用のスタイルシート（`css/hm-lp.css`）の内容を WordPress に追加します。

### 手順

1. 管理画面 左メニュー **外観 → カスタマイズ** をクリック
2. カスタマイザーが開いたら **「追加CSS」** をクリック
3. `css/hm-lp.css` ファイルをテキストエディタ（メモ帳など）で開く
4. ファイルの内容を**全部コピー**（Ctrl+A → Ctrl+C）
5. カスタマイザーの追加CSSボックスに**貼り付け**（Ctrl+V）
6. 右上の「公開」ボタンをクリックして保存

> **SWELL の場合:** 外観 → カスタマイズ の代わりに、  
> SWELL設定 → 「カスタムCSS / JS」タブ にある追加CSSボックスでも可能です。

## 4. JavaScript を導入する

JavaScript はページのアニメーション（タイトルドロップイン、パーティクル、スクロールアニメ、アコーディオン等）に必要です。
以下の **A または B** どちらかの方法で導入してください。

> **ファイルについて:** `js/hm-lp.js` 1ファイルにすべての機能を統合しています。
> `script.js` は不要です（内容は `hm-lp.js` に統合済み）。

---

### 方法 A：functions.php に追記する（推奨）

1. `js/hm-lp.js` ファイルを、お使いの**子テーマ**の `js/` フォルダにアップロードする  
   （FTPソフトまたはサーバーのファイルマネージャーを使用）  
   アップロード先の例: `/wp-content/themes/子テーマ名/js/hm-lp.js`

2. 子テーマの `functions.php` をテキストエディタで開く  
   （管理画面 **外観 → テーマファイルエディタ** → functions.php でも編集可能）

3. ファイルの**末尾**に以下のコードを追記して保存する:

```php
function hm_lp_scripts() {
    wp_enqueue_script(
        'hm-lp-js',
        get_stylesheet_directory_uri() . '/js/hm-lp.js',
        array(),
        '1.0.0',
        true
    );
}
add_action( 'wp_enqueue_scripts', 'hm_lp_scripts' );
```

> `true` の引数はスクリプトをフッターに読み込む設定です。DOMContentLoaded での実行を保証します。

---

### 方法 B：Code Snippets プラグインを使用する

コードを直接 PHP ファイルに書きたくない場合は、  
「**Code Snippets**」または「**WP Code**」プラグインをインストールして使用できます。

1. 管理画面 **プラグイン → 新規追加** で「Code Snippets」を検索・インストール・有効化
2. 左メニュー **スニペット → 新規追加** をクリック
3. タイトルに「HM LP Scripts」と入力
4. コードボックスに上記の PHP コードを貼り付ける
5. 「すべてのページで実行」を選択し、「保存して有効化」をクリック

> **注意:** 方法 B の場合も `js/hm-lp.js` を子テーマの `js/` フォルダにアップロードする必要があります。

## 5. 固定ページを作成する

LP用の固定ページを作成し、各セクションをブロックとして追加します。

### 固定ページの作成手順

1. 管理画面 **固定ページ → 新規追加** をクリック
2. タイトルを「悪魔が来たりて、ヘヴィメダル」などに設定
3. URLスラッグを `hm-lp` などに設定（パーマリンク設定）
4. テンプレートを「**ブランク（ヘッダー・フッターなし）**」または「**LP用テンプレート**」に設定（SWELLの場合）

### ブロック配置順序

以下の順番で**カスタムHTMLブロック**を追加し、各 `html/` ファイルの内容を貼り付けてください。

| 順番 | ファイル名 | 内容 |
|---|---|---|
| 1 | `html/block-01-hero.html` | ヒーローセクション |
| 2 | *(カスタムHTML または SWELLブロック)* | 企画概要（about）セクション |
| 3 | *(カスタムHTML または SWELLブロック)* | 対象機種（machine）セクション |
| 4 | *(カスタムHTML または SWELLブロック)* | 兎味ペロリナ（pelorina）セクション |
| 5 | `html/block-02-schedule.html` | 開催スケジュール |
| 6 | `html/block-03-wave-results.html` | WAVE 結果アコーディオン |
| 7 | `html/block-04-finalists.html` | 最後の審判 進出店舗 |
| 8 | `html/block-05-banners.html` | バナー広告枠 |

> **企画概要・対象機種・ペロリナセクション**については、`preview.html` 全体の該当部分をカスタムHTMLブロックに貼り付けるか、SWELLのリッチテキスト・画像ブロックで再現してください。

### カスタムHTMLブロックの追加方法

1. ブロックエディタ（Gutenberg）で「+」ボタンをクリック
2. 「カスタムHTML」と検索してブロックを追加
3. HTMLコードボックスに各ファイルの内容を貼り付ける

## 6. SWELL 投稿リストブロックの設定

WAVE結果セクションの各アコーディオンに、対応するWAVEの結果記事を自動で表示するには、  
記事にタグを設定して SWELL の投稿リストブロックで絞り込む方法が便利です。

### WAVEごとのタグ設定

| WAVE | タグ名 | 用途 |
|---|---|---|
| WAVE 1 | `hm-wave1` | 第一の騎士達 結果記事 |
| WAVE 2 | `hm-wave2` | 第二の騎士達 結果記事 |
| WAVE 3 | `hm-wave3` | 第三の騎士達 結果記事 |
| WAVE 4 | `hm-wave4` | 第四の騎士達 結果記事 |
| WAVE 5 | `hm-wave5` | 第五の騎士達 結果記事 |

### 記事にタグを付ける手順

1. 結果レポート記事を投稿・編集する
2. 右サイドバーの「タグ」ボックスに対応するタグ（例: `hm-wave1`）を入力して Enter
3. 記事を公開・更新する

### SWELL 投稿リストブロックでタグ絞り込みをする方法

現在の LP では `block-03-wave-results.html` のアコーディオン内に記事リンクを手動で記載しています。  
WAVE終了後、以下のいずれかの方法で記事を更新してください：

**方法①（手動更新）:** `block-03-wave-results.html` 内の `<li><a href="...">` 部分に実際の記事URLとタイトルを入力する

**方法②（SWELL投稿リストブロック）:**
1. カスタムHTMLブロック内のアコーディオン `<div class="hm-accordion-body">` の中に、SWELLの「投稿リストブロック」を配置
2. ブロック設定で「タグで絞り込む」を選択し、対応するタグ（例: `hm-wave1`）を指定
3. 表示件数や表示形式をLP用に調整する

## 7. 動作確認チェックリスト

ページ公開後、以下の項目を確認してください。

- [ ] **ヒーローアニメーション:** ページを開いたとき、タイトル文字が上から1文字ずつ落ちてくる
- [ ] **スクロールロック:** タイトルアニメーション中はスクロールができない（完了後に解除される）
- [ ] **ヒーロー要素フェードイン:** タイトル完了後、サブタイトル・期間・筐体画像がフェードインする
- [ ] **キャッチコピーアニメ:** 企画概要セクションにスクロールしたとき、テキストが行ごとにフェードインする
- [ ] **WAVEアコーディオン:** WAVE 1〜5 のボタンをクリックすると開閉できる
- [ ] **アコーディオン自動展開:** ページ読み込み時、記事がある最新WAVEが自動で開いている
- [ ] **スケジュールの矢印アニメ:** 各WAVEの間に小さな下矢印がゆらゆら動いている
- [ ] **ハンバーガーメニュー（スマホ）:** スマホ表示でメニューボタンをタップすると全画面ナビが開く
- [ ] **画像の表示:** すべてのセクションで画像が正常に表示されている（404エラーなし）
- [ ] **レスポンシブ対応:** スマホ・タブレット・PCで各セクションのレイアウトが崩れていない
